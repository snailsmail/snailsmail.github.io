Common Command Recording

sass

Sass,英文全称"Syntactically Awesome StyleSheets",是对CSS的扩展.它添加了一些CSS语法不具
有的新特性.Sass让开发者维护样式变的更加简单.
Sass完全兼容CSS,因为它是一个预处理器.它用Sass语法写代码,然后转换成标准的CSS.它允许使用变量,
嵌套规则,导入等众多功能.它有助于保持大型样式表结构良好,让css语言更加强大,优雅.
Sass有两种语法,第一种被成为SCSS(Sassy CSS),它是CSS语法的扩充版本,以后缀名为.scss的文件也是
合法的CSS样式表文件.
第二种是比较古老的语法.就是所说的缩排语法(直接称为"Sass").它不是用花括号,而是通过缩排的方式表达
选择符的嵌套层级.也不使用分号,而是用换行符来分隔属性,扩展名名为.sass.

用Sass变量存储数据

Sass不同于CSS的一个特点就是它允许使用变量,js中用let和const关键字声明变量,在Sass中使用$开头
开声明变量,后面紧跟变量名.
例如:当多个元素需要相同的颜色,就可以声明变量,一旦需要更改颜色,只需要修改这个变量值就好.

用Sass嵌套CSS

Sass允许CSS规则的嵌套,这在写样式的时候会很有用,嵌套写法即在对应的父元素中写子元素的样式.

用Mixins创建课重用的CSS

在Sass中,mixin是一组css声明,可以在整个样式表中重复使用.
CSS的新功能需要一段时间适配后,所有浏览器后才能完全使用.随着浏览器的不断升级,使用这些CSS规则时可
能需要添加浏览器前缀.如:'box-shadow'
<css code>
  div {
    -webkit-box-shadow: 0px 0px 4px #fff;
    -moz-box-shadow: 0px 0px 4px #fff;
    -ms-box-shadow: 0px 0px 4px #fff;
    box-shadow: 0px 0px 4px #fff;
  }
</>
对于所有具有box-shadow属性的元素重写此规则,或者更改每个值以测试不同的效果,需要花费大量精力.
Mixins就像CSS的函数,例如:
@mixin box-shadow ($x, $y, $blur, $c) {
    -webkit-box-shadow: $x $y $blur $c;
    -moz-box-shadow: $x $y $blur $c;
    -ms-box-shadow: $x $y $blur $c;
    box-shadow: $x $y $blur $c;
}
定义以@mixin开头,后面跟自定义名称,现在,只需要在需要的地方使用@include调用上面定义的mixin,就可以
自动添加好所有浏览器前缀, mixin使用@include指令调用:
div {
  @include box-shadow(0px, 0px, 4px, #fff);
}

使用@if和@else为你的样式添加逻辑

Sass中的@if指令对于测试特定情况非常有用.于js中的if语句类似,例如:
@mixin make-bold($bool) {
  @if @bool == true {
    font-weight: bold;
  }
}
或者添加更多条件:
@mixin text-effect($val) {
  @if $val == danger {
    color: red;
  }
  @else if $val == alert {
    color: yellow;
  }
  @else if $val == success {
    color: green;
  }
  @else {
    color: black;
  }
}

使用@for创建一个Sass循环

你可以在Sass中使用@for,它于js中的for循环类似.
@for以两种方式使用: 'start through end'或'start to end',主要区别在于'开始结束'排除
结束号码,而'开始结束'包括结束号码
例如:
@for $i from 1 through 12 {
  .col-#{$i} {width: 100%/12 * $i}
}
#{$i}部分是将变量i与文本组成字符串的语法,这是创建网格布局最有效的方法.

使用@each遍历列表中的项目

在每次迭代时,变量将从列表映射赋值给当前值
@each $color in blue, red, green {
  .#{$color}-text {color: $color}
}
map的语法略有不同,例如:
$colors: {color1: blue, color2: red, color3: green}
@each $key, $color in $colors {
  .#{$color}-text {color: $color}
}
注意,需要$key变量来引用map中的键,否则,编译后的CSS将包含color1,color2...

使用@while当条件满足时样式生效

Sass中的@while指令与js中的while类似,只要满足条件,他就会一直创建CSS代码
$x: 1;
@while $x < 13 {
  .col-#{$x} {width: 100%/12*$x}
  $x: $x + 1;
}

用Partials将你的样式分成小块

Sass中的Partials是包含CSS代码段的单独文件,这些是在其他Sass文件中导入和使用的,我们可以
把类似的代码放在模块中,以保持代码结构规整且易于管理
partials的名称以下划线_字符开头,然后告诉Sass它是CSS的一小部分,而不是将其转换成为CSS文件,
此外,Sass文件以.scss文件扩展名结尾,要将partial中的代码放入另一个Sass文件中,请使用@import
指令例如,如果所有mixins都保存在名为'_mixins.scss'的partial中,并且在'main.scss'文件中
需要他们,使用@import 'mixins'将外部文件导入,注意,import语句中不需要下划线,Sass知道它
是partial.将partial导入文件后,可以使用所有变量mixins和其他代码

将一组CSS样式扩展到另一个元素

Sass有一个名为extend的功能,可以很容易的从一个元素中借用CSS规则并在另一个元素上重用他们
例如:下面的CSS规则块设置了.panel class的样式,它有一些样式,现在你需要另一个名为.big-panel
的面板,它具有与.panel相同的基本属性,extend指令是一种重用为另一个元素编写的规则的简繁方法:
.panel{ height: 100px}
.big-panel {@extend .panel；width: 100px}
